while Loop

Tests condition BEFORE each iteration · minimum 0 executions
Syntax
while (condition) { // runs while condition is true // must update something so it // eventually becomes false }
Execution flow
1
Check condition
2
If TRUE → run code block → back to step 1
3
If FALSE → exit loop
When to use
  • You don't know how many iterations are needed
  • The loop depends on user input or changing conditions
  • The code might not need to run at all
Example — input validation
int age = -1; while (age < 0 || age > 120) { Console.Write("Enter age (0-120): "); age = int.Parse( Console.ReadLine()); } Console.WriteLine("Age: " + age);

do-while Loop

Tests condition AFTER each iteration · minimum 1 execution
Syntax
do { // always runs at least once // then checks condition } while (condition); // ← semicolon!
Execution flow
1
Run code block (always, first time)
2
Check condition
3
If TRUE → back to step 1
4
If FALSE → exit loop
When to use
  • Code must run at least once before testing
  • Menu systems — show menu, then ask if repeat
  • "Play again?" style game loops
Example — menu system
string choice; do { Console.WriteLine("1. View"); Console.WriteLine("2. Edit"); Console.WriteLine("3. Exit"); Console.Write("Choice: "); choice = Console.ReadLine(); } while (choice != "3");

for Loop

Exact iterations known · init · condition · increment in one line
Syntax
for (init; condition; increment) { // init runs ONCE at start // condition checked BEFORE each pass // increment runs AFTER each pass } // Example: count 1 to 10 for (int i = 1; i <= 10; i++) { Console.WriteLine(i); }
Execution flow
1
Initialize counter (once)
2
Check condition → if FALSE, exit
3
Run code block
4
Increment → back to step 2
When to use
  • You know exactly how many times to repeat
  • Counting sequences (1 to 100, 10 down to 1)
  • Processing a fixed number of items or students
Common for patterns
// Count up for (int i = 1; i <= 10; i++) // Count down for (int i = 10; i >= 1; i--) // Even numbers only for (int i = 2; i <= 20; i += 2) // Count by fives for (int i = 0; i <= 100; i += 5)

Common Mistakes to Avoid
⚠ Infinite Loop — Forgetting to Update the Counter

Inside a while or do-while, something must change during each iteration that will eventually make the condition false. A loop where the condition never becomes false will run forever and freeze the program.

⚠ Off-by-One Error — < vs <=

for (int i = 1; i < 10; i++) prints 1–9, not 1–10. Use <= when the boundary value should be included in the loop.

⚠ Stray Semicolon After Loop Header

for (int i = 1; i <= 10; i++); — that trailing semicolon ends the loop immediately. The code block in braces then runs only once, after the loop has already finished.
