Loop 1 — while
1

while Loop

Tests condition BEFORE each iteration  ·  minimum 0 executions
Syntax
while (condition)
{
    // Runs as long as condition is true.
    // Something inside must eventually
    // make the condition false, or the
    // loop will run forever.
}
Execution flow
1
Check condition
2
If TRUE, run the code block, then return to step 1
3
If FALSE, exit the loop and continue the program
When to use
  • You do not know how many iterations are needed in advance
  • The loop depends on user input or a changing condition
  • It is acceptable for the code block to never run at all
Example — input validation
int age = -1;

while (age < 0 || age > 120)
{
    Console.Write("Enter age (0-120): ");
    age = int.Parse(Console.ReadLine());

    if (age < 0 || age > 120)
    {
        Console.WriteLine("Invalid. Try again.");
    }
}

Console.WriteLine("Age accepted: " + age);

2

do-while Loop

Tests condition AFTER each iteration  ·  minimum 1 execution
Syntax
do
{
    // Always runs at least once.
    // The condition is checked after
    // the block executes, not before.
}
while (condition);   // ← semicolon required
Execution flow
1
Run the code block (always, on the first pass)
2
Check condition
3
If TRUE, return to step 1
4
If FALSE, exit the loop and continue the program
When to use
  • The code must execute at least once before any condition is tested
  • Menu systems where the menu must appear before asking if the user wants to continue
  • Game loops where the game plays once, then asks "play again?"
Example — menu system
string choice;

do
{
    Console.WriteLine("=== MAIN MENU ===");
    Console.WriteLine("1. View Account");
    Console.WriteLine("2. Make Deposit");
    Console.WriteLine("3. Exit");

    Console.Write("Enter choice: ");
    choice = Console.ReadLine();
}
while (choice != "3");

Console.WriteLine("Goodbye!");

3

for Loop

Exact iterations known  ·  init · condition · increment in one line
Syntax
for (init; condition; increment)
{
    // init       runs ONCE at the start.
    // condition  checked BEFORE each pass.
    // increment  runs AFTER each pass.
}

// Example: count from 1 to 10
for (int i = 1; i <= 10; i++)
{
    Console.WriteLine(i);
}
Execution flow
1
Initialize the counter variable (once only)
2
Check condition — if FALSE, exit the loop
3
Run the code block
4
Increment the counter, then return to step 2
When to use
  • You know exactly how many times the loop should run
  • Counting sequences such as 1 to 100, or 10 down to 1
  • Processing a fixed number of items or students
Common for patterns
// Count up from 1 to 10
for (int i = 1; i <= 10; i++)

// Count down from 10 to 1
for (int i = 10; i >= 1; i--)

// Even numbers: 2, 4, 6 ... 20
for (int i = 2; i <= 20; i += 2)

// Count by fives: 0, 5, 10 ... 100
for (int i = 0; i <= 100; i += 5)

Common Mistakes to Avoid
Mistake 1 — Infinite Loop: Forgetting to Update the Counter

Inside a while or do-while, something must change during each iteration that will eventually make the condition false. If nothing changes, the condition stays true forever and the program freezes.

Mistake 2 — Off-by-One Error: < vs <=

Writing i < 10 instead of i <= 10 causes the loop to stop one iteration early, printing 1 through 9 instead of 1 through 10. Always check whether the boundary value itself should be included.

Mistake 3 — Stray Semicolon After the Loop Header

Writing for (int i = 1; i <= 10; i++); ends the loop immediately with the semicolon. The code block in the braces then runs only once after the loop finishes, rather than repeating 10 times.
