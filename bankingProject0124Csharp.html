Project Instructions

Clickety Clank Bank has hired you to write a C# program that will allow customers to make transactions from their bank accounts. The bank’s program requires the ability for customers to make one of two types of deposits: cash only and change only. In addition, customers also need to have the ability to process withdrawals based on their current account balance. Once the transaction is completed, the denominations of the transaction along with the updated balance must be displayed to the user.  Follow the steps below to implement the logic of the program for Project 1.

Steps:

  1. Use Random Numbers to generate a customer’s current account balance as a double. Be sure that the account balance is between $1.00 and $5,000.00. Then display a welcome message containing the user’s current account balance along with a menu to offer the user a choice between the different types of transaction. For this Project, the bank only will allow customers to make deposits (Option “A”) or withdrawals (Option “B”). If the user enters a choice that is not on the main menu, display an error and immediately terminate the program. While the bank understands that developers desire to be creative with presentation and input, there are strict guidelines in place for design. Therefore, the welcome messaging and menu format must match the ones in the Sample Run Videos to stay in compliance with the organization’s standards and requirements. See Sample Run Video I for an example of how the main menu error messaging must look in your program.
    • If the user chooses to make a Deposit (Option “A”), the program must do the following:
      1. The program must display a deposit menu to offer the user a choice between different types of deposits. At this bank, customers can make one of the following types of deposits: Cash Only (Option “A”) or Change Only (Option “B”). If the user enters a choice that is not on the deposit menu, display an error and immediately terminate the program. If the program is terminated, display a transaction receipt that shows no deposit was processed. See Sample Run Video E for an example of how an invalid deposit menu choice must look. To meet the requirements for the Bank, the deposit menu must match the ones in the Sample Run Videos. See Sample Run Video F for an example of how the deposit menu must look in your program.
      2. If the user chooses to make a Cash Only deposit (Option “A”), the program must do the following: 
        • Ask the user to input the number of $100.00 bills (hundreds), $50.00 bills (fifties), $20.00 bills (twenties), $10.00 bills (tens), $5.00 bills (fives), and $1.00 bills (ones). Clickety Clank Bank can only accept up to 100 bills of any type at a time. Therefore, if a user enters a negative amount or more than 100 of any type of bill, display an error message and process the input of the number of that type of bill as 0. For example, if a user enters 125 fifty-dollar bills, display an error message for that input and process the transaction with 0 fifty-dollar bills entered. See Sample Run Videos A, B, C, and D for an example of how the inputs of a Cash Only deposit (Option “A”) must look.
      3. If the user chooses to make a Change Only deposit (Option “B”), the program must do the following: 
        • Ask the user to input the number of quarters ($0.25), dimes ($0.10), nickels ($0.05), and pennies ($0.01). Clickety Clank Bank can only accept up to 100 coins of change of any type at a time. Therefore, if a user enters a negative amount or more than 100 of any type of coin, display an error message and process the input of the number of that type of coin as 0. For example, if a user enters -88 dimes, display an error message for that input and process the transaction with 0 dimes entered. See Sample Run Videos E, F, G, and H for an example of how the inputs of a Change Only deposit (Option “B”) must look.
      4. Once a valid deposit (Cash only or Change only) has been made, the program must use named constants that hold each dollar and coin amount to calculate the amount of money to be deposited to the account based on the value of each dollar and / or coin amount. Then, display a transaction receipt that contains the customer's starting balance, the deposit amount (based on the amounts of dollar bills or coins deposited), a deposit breakdown that shows the number of each type of dollar bill for a cash deposit or the number of each type of coin for a change deposit along with the dollar amount of each unit. If an amount of a type of dollar bill or coin equals 0, do not display the unit in the deposit breakdown. See Sample Run Video A and Sample Run Video F for an example of how transaction receipt with a cash only deposit breakdown looks. See Sample Run Video B and Sample Run Video G for an example of how transaction receipt with a cash only deposit breakdown looks.
    • If the user chooses to make a Withdrawal (Option “B”), the program must do the following:
      1. The program must display the user’s current balance and ask the user how much money to withdraw from the bank account account. At Clickety Clank Bank, users cannot withdraw more than what is available, nor can they request withdrawals of negative amounts. If the customer requests an invalid withdrawal amount, display an error and immediately terminate the program. If the program is terminated, display a transaction receipt that shows no withdrawal was processed. See Sample Run Video H for an example of how an invalid withdrawal request must look.
      2. If a valid withdrawal request is made, the program must use a series of sequential division and modular division calculations to calculate the amount of hundred-dollar bills (hundreds), fifty-dollar bills (fifties), twenty-dollar bills (twenties), ten-dollar bills (tens), five-dollar bills (fives), one-dollar bills (ones), quarters ($0.25), dimes ($0.10), nickels ($0.05), and pennies ($0.01). Then, the program must use named constants that hold each dollar and coin amount to calculate the amount of money to be withdrawn from the account based on the number of each dollar and / or coin.
      3. Finally, display a transaction receipt that contains the customer's starting balance, the withdrawal amount (based on the amounts of dollar bills or coins calculated in the withdrawal), a withdrawal breakdown that shows the number of each type of dollar bill and or/ coin for a withdrawal along with the dollar amount of each unit. If an amount of a type of dollar bill or coin equals 0, do not display the unit in the deposit breakdown. See Sample Run Video C for an example of how transaction receipt with a withdrawal breakdown looks.
  2. Your program's output MUST be formatted in the same manner as the Sample Runs.
  3. Save your .cpp file using the required naming format.
  4. Click the assignment link to upload your file.
